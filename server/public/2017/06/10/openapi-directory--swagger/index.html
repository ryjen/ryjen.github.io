OpenAPI Directory & Swagger

Was browsing github and I came across the OpenAPI Directory. A good directory of API’s will help in thinking of a mobile app idea.

Couldn’t help but notice that all the API’s implement something called OpenAPI ("Swagger") 2.0. Ok, great, consistency in API specs, that will help if you had to write one.

In fact, most web apps have been moving towards an architecture of an API backend and Javascript frontend that calls the API, so chances are if you had to write a web app it will have an API.

Looks like I’m behind the ball again (eh), as there are already some cool tools that take advantage of this API consistency.

  • commandcar is a CLI tool that can install OpenAPI specs, then run queries against them.

  • paw is a very nice looking macOS app that can install API templates for doing the same.

  • go-swagger is a tool to generate an implementation of your api spec

  • dredd is a tool for testing your swagger api against an implementation

Long story short, defining an API in SwaggerHub or with Swagger is quite nice for consistency and the tooling support is great. Quite enjoying my fast performant api backend for a backbone or mobile frontend.

NOTE: For simpler use cases you can probably spin something on firebase or any other backend-as-a-service.
